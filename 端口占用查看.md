端口占用查看

## 使用 `netstat` 命令

```
# 查看所有监听端口和对应的进程
netstat -tulpn

# 查看特定端口（如 80 端口）
netstat -tulpn | grep :80

# 查看所有 TCP 连接
netstat -tpn

# 查看所有 UDP 连接
netstat -upn
```



**参数说明：**

- `-t`：TCP 端口
- `-u`：UDP 端口
- `-l`：仅显示监听端口
- `-n`：显示数字格式（不解析主机名和服务名）
- `-p`：显示进程信息

## 使用 `ss` 命令（推荐，更现代）

```
# 查看所有监听端口
ss -tulpn

# 查看特定端口
ss -tulpn | grep :80

# 查看所有 TCP 连接
ss -tnp

# 查看所有 UDP 连接
ss -unp
```



**`ss` 比 `netstat` 更快，是推荐使用的工具。**

## 使用 `lsof` 命令

```
# 查看所有网络连接
lsof -i

# 查看特定端口
lsof -i :80

# 查看 TCP 连接
lsof -i tcp

# 查看 UDP 连接
lsof -i udp

# 查看端口范围
lsof -i :1-1024
```



## 使用 `fuser` 命令

```
# 查看特定端口被哪个进程使用
fuser 80/tcp
fuser 443/udp

# 显示详细信息
fuser -v 80/tcp
```



## 实用示例

### 查找 SSH 端口占用

```
# SSH 默认端口 22
ss -tulpn | grep :22
# 或者
netstat -tulpn | grep :22
# 或者
lsof -i :22
```



### 查找 HTTP/HTTPS 端口占用

```
# 80 端口 (HTTP)
ss -tulpn | grep :80

# 443 端口 (HTTPS)  
ss -tulpn | grep :443
```



### 查找 MySQL 端口占用

```
# MySQL 默认端口 3306
ss -tulpn | grep :3306
```



## 6. 查看具体进程信息

找到占用端口的进程 ID (PID) 后，可以查看进程详情：

```
# 查看进程信息
ps aux | grep [PID]

# 或者使用
cat /proc/[PID]/status

# 查看进程打开的文件
lsof -p [PID]
```



## 7. 常用组合命令

```
# 一键查看所有监听端口及进程
sudo ss -tulpn

# 查看指定端口的详细信息
sudo lsof -i :8080

# 实时监控端口连接
watch -n 1 'ss -tulpn'
```



## 实际应用示例

```
# 查看 80 端口被谁占用
sudo ss -tulpn | grep :80

# 输出示例：
# tcp   LISTEN 0      128          0.0.0.0:80        0.0.0.0:*    users:(("nginx",pid=1234,fd=6))
```



**最推荐使用 `ss -tulpn`**，因为它速度快、信息全，是现代 Linux 系统的标准工具。

记住，查看监听端口通常需要 **root 权限**，所以在命令前加 `sudo`。
